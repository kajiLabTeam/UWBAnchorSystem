# 進捗報告 - UWBセンシングシステム開発

**報告日**: 2025年7月8日  
**進捗期間**: Nearby Connections API実装フェーズ

## 1. 研究目標と進捗概要

### 研究目標
- **UWBセンシングシステムの自動化**：複数端末での同期センシングと中央制御システムの構築
- **リアルタイム軌跡表示システム**：センシングデータのリアルタイム可視化による即座のフィードバック
- **研究実験効率化プラットフォーム**：センシング→データ収集→解析のワークフローを自動化

### 今回の進捗要約
Mac-Android間でのP2P通信基盤を完成させ、**研究実験におけるデータ収集の完全自動化**を実現。これまで手動で行っていた複数端末の制御とデータ回収作業を、中央制御システムから一斉操作可能にした。

## 2. 実装した研究支援機能

### 2.1 中央制御型センシングシステム
- **一斉センシング制御**：Mac側からコマンド一つで複数Android端末の同期センシングを開始/終了
- **リアルタイム監視**：各端末のセンシング状況をリアルタイムで可視化し、実験の進行状況を即座に把握
- **自動データ回収**：センシング終了と同時にCSVファイルを自動的にMacに転送・保存

### 2.2 リアルタイム軌跡表示機能
- **即座のフィードバック**：elevation、azimuth、distance、RSSIの数値をリアルタイム表示
- **実験品質管理**：センシング中にデータ品質（NLOS状況、信号強度）を監視可能
- **複数端末同時監視**：3列グリッド表示で最大9台の端末状況を同時確認

### 2.3 研究データ管理システム
- **自動ファイル命名**：`yyyyMMdd_HHmmss_[端末名]_[実験名].csv`形式で実験データを体系的に管理
- **メタデータ保持**：デバイス名、実験開始時刻、ファイルサイズなどを自動記録
- **即座のデータアクセス**：受信完了と同時にFinderでファイル表示、即座の解析作業が可能

## 3. 研究効率化への寄与

### 3.1 実験ワークフローの改善
**従来の手動プロセス**：
1. 各端末で個別にセンシングアプリを起動
2. 手動で開始タイミングを合わせる
3. 各端末からSDカードやUSBでファイル回収
4. ファイル名を手動で整理・管理

**新システムでの自動化プロセス**：
1. Mac側でファイル名入力→センシング開始ボタン
2. 全端末が自動的に同期してセンシング開始
3. リアルタイムでデータ品質を監視
4. センシング終了と同時に全ファイルがMacに自動転送・整理

**効率化効果**：実験準備時間を約80%短縮、データ回収時間を約90%短縮

### 3.2 データ品質向上
- **同期精度向上**：ネットワーク経由での一斉制御により、手動操作による時間差を排除
- **リアルタイム品質監視**：センシング中に異常値やNLOS状況を即座に検出
- **データ欠損防止**：自動転送により、SDカード取り忘れや端末故障によるデータ損失を防止

## 4. 技術的革新点

### 4.1 異種OS間リアルタイム通信
- **Nearby Connections API**：Android-macOS間でのP2P通信を実現
- **JSONベースプロトコル**：`REALTIME_DATA`、`SENSING_START`、`SENSING_STOP`などの構造化コマンド
- **ファイル直接転送**：ネットワーク経由でのバイナリファイル転送

### 4.2 スケーラブルアーキテクチャ
- **1対多通信**：Mac 1台で最大9台のAndroid端末を同時制御
- **状態管理**：各端末の接続状況、センシング状態、データ送信状況を個別管理
- **エラーハンドリング**：通信障害時の自動リトライとバックアップシステム
